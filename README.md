# Бот ГСУ СК — связь Discord и портала «Следак»

Бот связывает каналы Discord-сервера проекта с порталом «Следак» через защищённый HTTP API. Работает в обе стороны: читает сообщения из каналов и передаёт события на сайт, а также публикует данные из очереди портала обратно в Discord (дела, состав, отчёты ПГСкО, взыскания).

Бот не модерирует сервер, не удаляет чужие сообщения и не выдаёт роли. Делает ровно то, что описано ниже.

## Что умеет

**Из Discord в портал:**
- слушает шесть каналов проекта (распределение дел, дела, состав, взыскания, отпуска, внутренний оборот);
- разбирает сообщения (в том числе embed от форм-бота) и отправляет события в `/api/bot`: новые дела, смена статусов, состав, взыскания, отпуска;
- распознаёт кадровые приказы-картинки из «внутреннего оборота» (OCR) и вносит назначения/звания/увольнения;
- отпуска уходят в базу только после одобрения реакцией ✅.

**Из портала в Discord:**
- по заданию из очереди портала публикует дело в «дела-ск» нужным статусом (возбуждено / передано в прокуратуру / отказ / прекращение);
- по реакции прокуратуры ✅ переносит дело в архив;
- публикует и обновляет актуальный состав в «состав-ск»;
- публикует отчёты ПГСкО в канал проверки и по ✅ засчитывает их;
- публикует уведомления о взысканиях и карточки актов по делам.

## Как устроена связь

Бот не имеет прямого доступа к Neon Postgres:

- **бот → сайт**: распознанные события уходят на `/api/bot` с `BOT_API_SECRET`, HMAC-подписью, timestamp и nonce;
- **сайт → бот**: бот опрашивает `/api/bot`, получает очередь публикаций, публикует в Discord и отправляет подтверждения обратно.

Google-таблиц и Apps Script в схеме больше нет — всё через Postgres.

## Запуск

### Docker (для сервера)

```bash
cp .env.example .env     # заполнить токен, BOT_API_URL/BOT_API_SECRET и каналы/OCR
docker compose up -d     # сборка и запуск в фоне, автоперезапуск
docker compose logs -f   # логи
```

Остановить: `docker compose down`. Логи лежат в `./logs` (примонтированы томом, переживают пересоздание контейнера).

### Node (для локальной проверки)

```bash
npm install
npm run check:env   # проверка переменных
npm start
```

Проверка парсеров без токена — `npm run dry-run`. Проверка синтаксиса — `npm run syntax`.

## Переменные окружения

Скопируй `.env.example` в `.env` и заполни. Ключевые:

| Переменная | Назначение |
|---|---|
| `DISCORD_BOT_TOKEN` | токен бота |
| `DISCORD_CHANNEL_IDS` | пусто на бою (слушает каналы из `channelRules.js`); список ID — для теста |
| `BOT_API_URL` | endpoint сайта, обычно `https://sledak-rmrp.ru/api/bot` |
| `BOT_API_SECRET` | общий секрет с Vercel-переменной портала; **обязателен** |
| `BOT_UNIT` | управление этого экземпляра бота: `arbat`, `rublevka`, `patriki`, `tverskoy`, `kutuzovsky` или `ca`; **обязателен** |
| `REPORT_CHANNEL_ID` | ID канала еженедельного отчёта |
| `PGSKO_REPORT_CHANNEL_ID` | ID канала проверки отчётов ПГСкО |
| `ACT_REVIEW_CHANNEL_ID` | ID канала «акты-и-делоодобрение» |
| `PGSKO_APPROVER_ROLE_ID` | ID роли, чья ✅ засчитывает ПГСкО; без него реакции игнорируются |
| `PROSECUTOR_ROLE_ID` | ID роли прокуратуры (для публикаций и архивации по ✅) |
| `OCR_API_KEY` | ключ aitunnel для OCR приказов (пусто — OCR выключен) |
| `LOG_RAW_MESSAGES` | `false` на бою; `true` только для временной диагностики сырых Discord-снапшотов |
| `DISCORD_ENABLE_GUILD_MEMBERS` | `true`, если включён Server Members Intent (для @упоминаний в составе) |

Токены и ключи хранятся только в `.env`, который не попадает в репозиторий.

## Интенты и права

Интенты: `Guilds`, `GuildMessages`, `MessageContent`, `GuildMessageReactions`, `GuildMembers`.

`MessageContent` и `GuildMembers` — привилегированные, включаются в Developer Portal (Bot → Privileged Gateway Intents). `MessageContent` нужен, чтобы видеть текст сообщений; `GuildMembers` — чтобы упоминать сотрудников по нику при публикации состава.

Права на каналах: `View Channel`, `Read Message History`, `Send Messages`, `Add Reactions`. Модераторские права (`Administrator`, `Manage Messages`, `Manage Roles`, `Ban`, `Kick`) не нужны.

Как создать бота, токен и включить интенты — в `integration/DISCORD-BOT-SETUP.md`.

## Каналы

| Канал | Что делает бот |
|---|---|
| `распределение-дел-ск` | заносит новое дело в базу |
| `дела-ск` | обновляет статус дела по номеру; публикует дела из базы |
| `состав-ск` | разбирает состав в базу; публикует состав из базы |
| `аудит-взысканий` | пишет предупреждения/выговоры сотрудникам |
| `рапорта-на-отпуск` | меняет статус сотрудника после одобрения ✅ |
| `внутренний-оборот` | распознаёт приказы-картинки (OCR) и вносит кадровые изменения |

ID каналов — в `src/channelRules.js`.

## Логи

В папке `logs` (права 700 на папку, 600 на файлы):
- `messages.ndjson` — нормализованные события без сырого снапшота Discord по умолчанию;
- `raw-messages.ndjson` — сырой снимок всех полей сообщения Discord, только если `LOG_RAW_MESSAGES=true`;
- `messages.csv` — плоская таблица для просмотра;
- `sheet-actions.ndjson` — только распознанные действия.

## Команды

Бот не обрабатывает команды пользователей. Он работает автоматически: по событиям в каналах и по заданиям из очереди `outbox_events`. Это уменьшает поверхность атаки и упрощает проверку.

## Используемые библиотеки

- `discord.js` — подключение к Discord Gateway, обработка событий и отправка сообщений;
- `@neondatabase/serverless` — доступ к базе Neon Postgres;
- `dotenv` — чтение переменных окружения из `.env`.

## Внешние сервисы

Бот обращается к двум внешним сервисам исходящими HTTPS-запросами, без приёма входящих подключений:

- **портал «Следак»** (`/api/bot`) — бот отправляет события и читает очередь публикаций;
- **aitunnel** (`api.aitunnel.ru`, OpenAI-совместимый API) — только для OCR кадровых приказов-картинок. Если `OCR_API_KEY` не задан — сервис не используется, картинки только логируются.

## Безопасность

- токены и ключи только в переменных окружения, не в коде;
- `/api/bot` дополнительно проверяет HMAC-подпись, timestamp и nonce;
- бот не запрашивает модераторских прав, не обрабатывает команды пользователей, не меняет роли, каналы и настройки сервера;
- полученные сообщения и действия сохраняются в локальные логи для проверки.
