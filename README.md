# Бот ГСУ СК — связь Discord и таблицы

Бот связывает каналы Discord-сервера проекта с рабочей Google-таблицей «ГСУ СК России по АФО». Работает в обе стороны: читает сообщения из каналов и переносит данные в таблицу, а также публикует данные из таблицы обратно в Discord (дела и состав).

Бот не модерирует сервер, не удаляет чужие сообщения, не выдаёт роли и не хранит базу данных. Делает ровно то, что описано ниже.

## Что умеет

**Из Discord в таблицу:**
- слушает шесть каналов проекта (распределение дел, дела, состав, взыскания, отпуска, внутренний оборот);
- разбирает сообщения (в том числе embed от форм-бота) и формирует записи для таблицы: новые дела, смена статусов, состав, взыскания, отпуска;
- распознаёт кадровые приказы-картинки из «внутреннего оборота» (OCR) и вносит назначения/звания/увольнения;
- отпуска уходят в таблицу только после одобрения реакцией ✅.

**Из таблицы в Discord:**
- по кнопке-чекбоксу публикует дело в «дела-ск» нужным статусом (возбуждено / передано в прокуратуру / отказ / прекращение);
- по реакции прокуратуры ✅ переносит дело в архив;
- по команде из меню публикует и обновляет актуальный состав в «состав-ск».

## Запуск

### Docker (для сервера)

```bash
cp .env.example .env     # заполнить токен, секрет, URL таблицы, ключ OCR
docker compose up -d     # сборка и запуск в фоне, автоперезапуск
docker compose logs -f   # логи
```

Остановить: `docker compose down`. Логи лежат в `./logs` (примонтированы томом, переживают пересоздание контейнера).

### Node (для локальной проверки)

```bash
npm install
npm run check:env   # проверка переменных
npm start
```

Проверка парсеров без токена — `npm run dry-run`. Проверка синтаксиса — `npm run syntax`.

## Переменные окружения

Скопируй `.env.example` в `.env` и заполни. Ключевые:

| Переменная | Назначение |
|---|---|
| `DISCORD_BOT_TOKEN` | токен бота |
| `DISCORD_CHANNEL_IDS` | пусто на бою (слушает каналы из `channelRules.js`); список ID — для теста |
| `OUTPUT_WEBHOOK_URL` | URL Apps Script Web App таблицы |
| `WEBHOOK_SECRET` | общий секрет, совпадает со Script Property в таблице |
| `PROSECUTOR_ROLE_ID` | ID роли прокуратуры (для публикаций и архивации по ✅) |
| `OCR_API_KEY` | ключ aitunnel для OCR приказов (пусто — OCR выключен) |

Токен и ключи хранятся только в `.env`, который не попадает в репозиторий.

## Интенты и права

Интенты: `Guilds`, `GuildMessages`, `MessageContent`, `GuildMessageReactions`, `GuildMembers`.

`MessageContent` и `GuildMembers` — привилегированные, включаются в Developer Portal (Bot → Privileged Gateway Intents). `MessageContent` нужен, чтобы видеть текст сообщений; `GuildMembers` — чтобы упоминать сотрудников по нику при публикации состава.

Права на каналах: `View Channel`, `Read Message History`, `Send Messages`, `Add Reactions`. Модераторские права (`Administrator`, `Manage Messages`, `Manage Roles`, `Ban`, `Kick`) не нужны.

## Каналы

| Канал | Что делает бот |
|---|---|
| `распределение-дел-ск` | заносит новое дело в «Дела в производстве» |
| `дела-ск` | обновляет статус дела по номеру; публикует дела из таблицы |
| `состав-ск` | разбирает состав в таблицу; публикует состав из таблицы |
| `аудит-взысканий` | пишет предупреждения/выговоры в лист «Состав» |
| `рапорта-на-отпуск` | меняет статус сотрудника после одобрения ✅ |
| `внутренний-оборот` | распознаёт приказы-картинки (OCR) и вносит кадровые изменения |

ID каналов — в `src/channelRules.js`.

## Логи

В папке `logs`:
- `messages.ndjson` — нормализованные события с полным снимком сообщения и результатом разбора;
- `raw-messages.ndjson` — сырой снимок всех полей сообщения Discord;
- `messages.csv` — плоская таблица для просмотра;
- `sheet-actions.ndjson` — только действия для таблицы.

Каждое действие также пишется в лист «Бот-лог» таблицы — это журнал для администрации.

## Интеграция с таблицей

Бот общается с таблицей через Apps Script Web App: отправляет распознанные действия POST-запросом и опрашивает очередь публикаций. Google-ключи боту не нужны — Web App работает от имени владельца таблицы. Развёртывание и приёмник — в папке `integration/`:

- `sheets-webapp.gs` — код Web App (приём действий, очередь публикаций, архивация);
- `DEPLOY.md` — как развернуть Web App;
- `DISCORD-BOT-SETUP.md` — как создать бота, токен и интенты;
- `REVERSE-PUBLISH-SETUP.md` — настройка публикации дел и состава.

## Команды

Бот не обрабатывает команды пользователей. Он работает автоматически: по событиям в каналах и по действиям из таблицы. Это уменьшает поверхность атаки и упрощает проверку.

## Используемые библиотеки

- `discord.js` — подключение к Discord Gateway, обработка событий и отправка сообщений;
- `dotenv` — чтение переменных окружения из `.env`.

Других зависимостей нет.

## Внешние сервисы (требуют согласования при размещении)

Для работы бот обращается к двум внешним сервисам — оба только исходящими HTTPS-запросами, без приёма входящих подключений:

- **Apps Script Web App** (`script.google.com`) — связь с рабочей Google-таблицей проекта: бот отправляет распознанные действия и забирает очередь публикаций. Запросы защищены секретным токеном.
- **aitunnel** (`api.aitunnel.ru`, OpenAI-совместимый API) — только для распознавания (OCR) кадровых приказов-картинок из «внутреннего оборота». Если `OCR_API_KEY` не задан — сервис не используется, картинки только логируются.

Базы данных, сторонних серверов и иных внешних подключений бот не использует.

## Безопасность

- токены и ключи только в переменных окружения, не в коде;
- база данных не используется;
- бот не запрашивает модераторских прав, не обрабатывает команды пользователей, не меняет роли, каналы и настройки сервера;
- весь обмен с таблицей защищён секретным токеном;
- полученные сообщения и действия сохраняются в локальные логи и в «Бот-лог» таблицы для проверки.
