# Развёртывание Apps Script Web App (приёмник для таблицы)

Цель: бот шлёт распознанные действия → Web App пишет их в таблицу
«ГСУ СК России по АФО». Google-ключи боту НЕ нужны — скрипт работает от
имени владельца таблицы.

## Шаг 1. Вставить код

1. Открой таблицу → **Расширения → Apps Script**.
2. Создай файл (например `WebApp.gs`) и вставь содержимое
   [`integration/sheets-webapp.gs`](./sheets-webapp.gs).
3. Сохрани проект (Ctrl/Cmd+S).

> Можно положить рядом с уже существующим кодом таблицы — функции не
> конфликтуют (`doGet`/`doPost` — отдельные точки входа).

## Шаг 2. Задать секрет

1. В редакторе Apps Script: **Настройки проекта (шестерёнка) → Свойства
   скрипта → Добавить свойство**.
2. Ключ: `WEBHOOK_SECRET`, значение: длинная случайная строка
   (например 32+ символа). Сгенерировать можно так:
   ```bash
   openssl rand -hex 24
   ```
3. Сохрани. **Этот же секрет пойдёт в `.env` бота (`WEBHOOK_SECRET`).**

## Шаг 3. Опубликовать как Web App

1. **Развернуть → Новое развёртывание → тип «Веб-приложение»**.
2. Параметры:
   - **Запуск от имени:** Я (владелец таблицы);
   - **Доступ:** «Все» (Anyone). Защиту обеспечивает секрет-токен, не Google-логин.
3. Нажми «Развернуть», выдай разрешения, скопируй **URL вида**
   `https://script.google.com/macros/s/AKfyc.../exec`.
4. Этот URL пойдёт в `.env` бота (`OUTPUT_WEBHOOK_URL`).

## Шаг 4. Проверка

**Живой ли сервис** (GET, без секрета — просто пинг):
```bash
curl -L "https://script.google.com/macros/s/ВАШ_ID/exec"
# {"ok":true,"service":"gsu-sk discord sink",...}
```

**Тестовое действие** (POST с секретом и токеном):
```bash
curl -L -X POST "https://script.google.com/macros/s/ВАШ_ID/exec?token=ВАШ_СЕКРЕТ" \
  -H "content-type: application/json" \
  -d '{"action":{"type":"raw_message","row":{"Текст":"тест из curl","Канал":"test"}},"meta":{"messageUrl":"http://example/1"}}'
# {"ok":true,"result":{"sheet":"Discord импорт","note":"raw"}}
```
После этого в таблице появятся листы **«Discord импорт»** (строка с тестом)
и **«Бот-лог»** (запись о действии). Если вернулось `{"ok":false,"error":"forbidden"}`
— не совпал секрет.

## Что куда (итог)

| Переменная бота (`.env`) | Значение |
|---|---|
| `OUTPUT_WEBHOOK_URL` | URL `…/exec` из шага 3 |
| `WEBHOOK_SECRET` | тот же секрет, что в Script Property |

## Поведение и ограничения (важно)

- **Взыскания пишутся в лист «Состав»** (колонки «Предупреждения»/«Выговоры»),
  отдельного листа «Аудит взысканий» больше нет. `discipline_event` ищет
  сотрудника по ФИО и обновляет только эти два числа; снятие/отработку
  взысканий нужно править в таблице вручную.
- ⚠️ После обновления кода Web App из `sheets-webapp.gs` нужно **переразвернуть**:
  «Развернуть → Управление развёртываниями → (карандаш) → Новая версия». URL
  при этом сохраняется. Также обнови Script Property `WEBHOOK_SECRET`, если
  секрет менялся.
- Web App пишет явные значения бота поверх старых формул-шаблонов в строке дела:
  данные из Discord важнее автозаготовок таблицы.
- Поля действия, которых нет среди заголовков листа, **игнорируются**
  (например «Дата события» для «Дел») — это безопасно.
- `case_status_event` обновляет статус дела по номеру. Перенос дела в архив
  выполняется при подтверждении (реакция ✅ от прокуратуры или чекбокс «В архив»).
- ПГСкО ведётся через Google Form, которую создаёт функция `setupPgSkOForm_()`.
  Ответы формы пишутся в лист «ПГСкО» со статусом «На проверке», публикуются
  ботом в Discord, а ✅ под сообщением меняет статус на «Зачтено».
- Каждое действие пишется в лист **«Бот-лог»** — это журнал для проверки.
