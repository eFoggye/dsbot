# Реверс-публикации (таблица → Discord): что доделать при выходе бота

Весь код готов и проверен синтаксисом. Осталось заполнить реальные ID и активировать.

## 1. Бот на сервере + интенты
- Добавить бота на боевой сервер.
- В Discord Developer Portal → Bot → включить **Server Members Intent** (privileged) — нужен для @плашек в составе.

## 2. Вытащить реальные ID (один раз)
```bash
npm run discover
```
Скрипт выведет (и сохранит в `logs/discover.txt`):
- кастомные эмодзи (`<:name:id>`) — для погон и значков ролей;
- роли (имя → id) — найти «[🔒] Прокуратура г. Москвы и МО» и роли должностей;
- список ников — сверить, что ник == ФИО в таблице.

## 3. Заполнить `src/publish/publishConfig.js`
- `RANK_EMOJI` — погоны по званиям (вставки `<:name:id>`).
- `POSITION_ROLE_IDS` — { "Должность": "roleId" } для @упоминания ролей должностей в составе (опционально; без них — должность текстом со значком).
- `PROSECUTOR_ROLE_ID` — лучше в `.env` (`PROSECUTOR_ROLE_ID=...`). Если задан — архивация по ✅ только от носителя роли; иначе любой ✅.
- При желании выверить `CASE_COLORS` (цвета полос) пипеткой по реальным сообщениям.

## 4. Apps Script проекта таблицы (ВАЖНО — два файла в одном проекте)
В Расширения → Apps Script должны лежать ОБА:
- **код таблицы v7** (с листа «Apps Script», ячейка A5) — onEdit (чекбоксы), меню «Опубликовать состав»;
- **WebApp.gs** (`integration/sheets-webapp.gs`) — doGet (очередь) / doPost (публикации, архив) + функции `enqueuePublish_`, `publishOnEdit_`, `publishRosterMenu_`, `archiveCaseByNumber_`.

Они вызывают функции друг друга — должны быть в одном проекте.
После вставки: **переразвернуть Web App** (Развернуть → Управление развёртываниями → Новая версия). URL сохранится.

## 5. Запуск
```bash
docker compose up -d
```

## 6. Обкатка
- В «Делах»: новое распределение приходит со статусом «Назначено».
- Выбрать статус «Возбуждено»/«Отказано в возбуждении» → поставить чекбокс **«Отправить»** → через ~15 сек в «дела-ск» появится embed, чекбокс снимется.
- После возбуждения можно публиковать «Прекращено»/«Передано в прокуратуру».
- «Приостановлено» остается только статусом в таблице и в «дела-ск» не публикуется.
- Прокуратура ставит ✅ под сообщением → дело уходит в «Архив дел». Либо чекбокс **«В архив»** в строке — перенос вручную.
- Меню **ГСУ СК — сервис → 📤 Опубликовать состав** → состав публикуется/обновляется в «состав-ск».

## Как это устроено (кратко)
Таблица (чекбокс/меню) → `enqueuePublish_` кладёт задание в скрытый лист «Очередь публикаций» → бот опрашивает Web App (`?mode=queue`, ~15 сек) → публикует в Discord → шлёт результат обратно (`case_published`/`roster_published`), Apps Script помечает задание готовым и сохраняет message_id. Архивация: бот ловит ✅ в «дела-ск», находит № дела по message_id (лист «Опубликованные дела») и переносит в архив.
